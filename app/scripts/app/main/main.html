What about that runtime?

This web app provides a way to run a Kevoree JavaScript node directly in your browser. It acts pretty much as the other Kevoree runtimes but considering that you are in a browser environment you will face some limitations, but will also be able to give your Kevoree component a UI frontend!

Browser limitations:

You cannot run any server in a browser. It means that you will have to use a Kevoree group that uses a centralized architecture. Real-time communications from a browser to some endpoint can be made using Web Sockets, though this runtime uses Kevoree's RemoteWSGroup by default.

The RemoteWSGroup allows nodes to share models using an external endpoint. We provide a public one at ws://ws.kevoree.org
