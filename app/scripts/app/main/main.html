What about that runtime?

This web app provides a way to run a Kevoree JavaScript node directly in your browser. It acts pretty much as the other Kevoree runtimes but considering that you are in a browser environment you will face some limitations, but will also be able to give your Kevoree component a UI frontend!

Browser limitations:

You cannot run any server in a browser. It means that you will have to connect this node platform to another Kevoree group. Because, it runs in the browser, a good way to communicate in real-time is by using Web Sockets, thought this runtime uses Kevoree's WSGroup by default.

Kevoree Web Editor:

In order to check if this runtime will be able to connect to another WSGroup, you can test the connection by using the Kevoree Web Editor and do:

File > Open from node

Enter your WSGroup connection information in the form, then click Pull model.
